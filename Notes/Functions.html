What are decorators for a function?

A decorator is a function that will take some other function as an argument, perform some operations and return some other function or the same function and all of this without changing any operability or structure of the function it accepted as an argument.

the following code gives an example of a wrapper function.

"""
This program will show how the wrapper functions are executed with the '@' symbol and what does
that mean.

"""


def outer_function(original_func):
    def inner_function(*args, **kwargs):
        print('the outer and inner functions are executed.')
        return original_func(*args, **kwargs)
    return inner_function
# when th outer_function will be executed it will not show any value as the return 
# for the outer function is inner_function that is not executed. 
# The output will be visible only after the inner_function() is executed which will
# in turn execute the original_function() with added arguments *args and **kwargs.

# this will define the function wrapper

@outer_function
def display():
    print('display function passed as an argument to outer_function is executed.')


@outer_function
def second_func(name, age):
    """
    This function will take two arguments and print those two arguments as name and age.
    :param name: my name
    :param age: my age
    :return: Nothing
    """
    print(f"My name is {name} and I am {age} years old.")

# This is same as saying that outer_function(display). and then when the display function is
# executed, it will execute the inner_function and this will also execute the outer function.
# The above syntax is exactly the same as display = outer_function(display)
# Now the question is what if our program has more than one original function and it takes more than
# one keyword argument? In this case we use *args or **kwargs.


display()
second_func('shubham', 26)

The following is the output for the code above. 

the outer and inner functions are executed.
display function passed as an argument to outer_function is executed.
the outer and inner functions are executed.
My name is shubham and I am 26 years old.

Similar to the functions, decorators can also be used with classes. The following code shows an example of the same.

class decorator_class(object):
    def __init__(self, original_function):
        self.original_function = original_function
        # What this will do, is that this will pass the original function to our class similar to
        # how it was passed to a function.

    def __call__(self, *args, **kwargs):
        print('call method was used to add functionality to the original function')
        return self.original_function(*args, **kwargs)


@decorator_class
def display():
    print('display function passed as an argument to decorator class.')


@decorator_class
def second_func(name, age):
    """
    This function will take two arguments and print those two arguments as name and age.
    :param name: my name
    :param age: my age
    :return: Nothing
    """
    print(f"My name is {name} and I am {age} years old.")

display()
second_func('shubham', 26)

The output of the code is 

call method was used to add functionality to the original function
display function passed as an argument to decorator class.
call method was used to add functionality to the original function
My name is shubham and I am 26 years old.
