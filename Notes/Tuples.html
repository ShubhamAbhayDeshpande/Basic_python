Tuples

Tuples are immutable lists. But this does not mean that we cannot perform any operations on the tuples.

The list of operations that can be performed on the tuples are listed in the python documentaion link below. The tuples are denoted by () brackets.

https://docs.python.org/3/library/stdtypes.html#common-sequence-operations

t = 'a', 'b', 'c'
print(t)

output

('a', 'b', 'c')

Notice that the python added the rackets in the output. But it is a good practice to add these brackets yourself in the code. Just to remove one additional thing to be remembered.

A single tuple can have values from multiple datatypes. The following has integer and string. 

print(("learning python", 2022))

output:

('learning python', 2022)

Since tuples are immutable they do not have overhead required to make changes. This saves memory. So if we have millions of data, tuples may save a lot of data.

The following code just shows what unpacking a sequence type means

#We can assign multiple values to the variables in the python. 

x,y,z = 1,4,6 # these values are actually tuples.

# Following code is what is referred to as unpacking a tuple
print(x)
print(y)
print(z)


# Unpacking a tuple

data = 1,4,6 # this is a tuple
x,y,z = data # The values on the left may look like a tuple but they are not a tuple. They are variables. And these variables get their values from the tupel data. 
# we can never have tuple on the left side of '=' because they are immutable. 
print(x)
print(y)
print(z)

The second advantage of tuple is that, since they are immutable, we can always know how many items are there to unpack. This may not always be the case with the list and other mutable sequence types.

The following code shows the nested list. It is a list containing the tuples and it is shown how we can unpack such lists.

albums = [("Welcome to my nightmare", "Alice Cooper", 1975),
("Bad Company", "Bad Company", 1974),
 ("Nightflight", "Budgie", 1984),
 ("More mayham", "Emilda may", 1981),
("Ride the lightening", "Metallica", 1984)]

item_1 = albums[0]
item_2 = albums[1]
item_3 = albums[2]
item_4 = albums[3]
item_5 = albums[4]


name= [item_1[0], item_2[0],item_3[0],item_4[0],item_5[0]]
album = [item_1[1], item_2[1],item_3[1],item_4[1],item_5[1]]
year = [item_1[2], item_2[2],item_3[2],item_4[2],item_5[2]]

for i in range(5):
    print("my discography:")
    print("band: {}, album: {}, year: {}".format(name[i],album[i],year[i]))

# The code below also gives the same output. But it saves us th effort of unpacking the individual tuples. 
# It is a more efficient way of writing the same code. 
for a, n, y in albums:
    print("band: {}, album: {}, year: {}".format(n,a,y))

The important choice at the time of using a nested dataset is when to use the tuples and the lists. It depends on the following:

  1. Are we planning to modify the dataset once it is written in the program
  1. what type of data are we planning to store? Heterogenous or homogenous data?
  1. How many levels of nesting are we planning to use?
